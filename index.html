制御構文とは

命令の実行順序を通常の逐次実行以外の順番に変化させる構文
Wikipedia - 制御構造より

(C言語での)真と偽について

C言語での真(true)は"0"以外

つまり偽は0のみ

なお論理式(後述)は
1が真、0が偽と明確に決められている。

処理(1)を実行して抜ける


                            if( 論理式(1) ){

else if文とelseは省略可能

この場合、論理式が偽であれば何も処理せずに抜ける

では、論理式とはどういうものなのか

論理式

そもそも論理式とは

論理式:論理演算を実行する式

では論理演算とは

先ほどの真と偽のどちらかのみを対象とした演算
(つまり結果が真か偽のみである処理)

for文

構文


						for(初期処理; 論理式; ループ時処理){
							処理;
						}

処理の順番

  1. 初期処理
  2. 論理式の評価
    • ここで偽であればfor文を抜けます
  3. 処理
  4. ループ時処理
  5. 2に戻る

初期処理

for文が開始される前に行われる処理

具体的にはここで変数の初期化などを行う

注意

(c99以前での)C言語では


						for(int i = 0; i < 100; i+)+

とは書けません

処理

好きな処理を書きましょう

ループ時処理

処理を行なった後に行う処理

値を順に増やしていきたいときなどはここに処理を書く

途中で抜けたいときはどうするの

-> break文を使いましょう

Sample Code


						for(i = 0; i < 100; i++){
							if(i == 2) break;
							printf("%d ", i);
						}

実行結果


						0 1

i = 2でbreakでif文の条件を満たして抜けているのでこれ以降の処理は行われない

じゃあ途中で飛ばしたいはどうするの

-> continue文を使いましょう

Sample Code


						for(int i = 0; i < 5; i++){
							if(i % 2 == 0) continue;
							printf("%d ", i);
						}

実行結果


						1 3

i % 2 == 0、つまりiが偶数の時に表示処理を飛ばしたということになる

switch 文

構文


						while(論理式){
							処理1;
						}

処理の流れ

  1. 論理式を評価する
    • 偽であればループを抜ける
  2. 処理1を実行する
  3. 1に戻る

論理式が真である間、処理1を実行し続ける

break, continueももちろん使える

while文では条件次第では1度も処理が行われないことも

それでは困るときはどうするのか？

do-while文

構文


						do{
							処理1;
						}while(論理式);

処理の流れ

  1. 処理1を実行する
  2. 論理式を評価する
    • 偽であればループを抜ける
  3. 1に戻る

処理と評価の順序が逆になっただけ

switch文

構文


						switch(変数i){
						case A:
							i == Aの時に行う処理
							break;
						case B:
							i == Bの時に行う処理
							break;
						case C:
							i == Cの時に行う処理
							break;
						.
						.
						default:
							上のどれにもiが当てはまらない時に行う処理
						}

switch文は以下の式と同等である


						if(i == A){
							i == Aの時に行う処理
						}else if(i == B){
							i == Bの時に行う処理
						}else if(i == C){
							i == Cの時に行う処理
						}
						.
						.
						}else{
							上のどれにもiが当てはまらない時に行う処理
						}

defaultはなくてもよい

ここでA,B,C...は整数値しか取れないことに注意

つまり


						switch(i){
						case 1.2:
							hogehoge;
						}

はコンパイルエラーとなる

注意点

breakが無いとどうなるのか


						if(i == A){
							i == Aの時に行う処理
							i == Bの時に行う処理
							i == Cの時に行う処理
							.
							.
							上のどれにもiが当てはまらない時に行う処理
						}

この動作をフォールスルー(Fall Through)と呼ぶ

意図して行うのならいいのだが、気付かずに行うと面倒なバグの原因となるので注意
